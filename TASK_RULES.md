# 태스크 실행 규칙

## 기본 규칙

1. **한 번에 하나의 태스크만 실행**
   - 태스크 목록에서 하나의 항목만 수행
   - 완료 후 반드시 멈춤

2. **허락 후 진행**
   - 각 태스크 완료 후 사용자 확인 대기
   - 사용자가 "다음", "진행", "OK" 등 승인 시에만 다음 태스크로 이동

3. **체크박스 업데이트**
   - 태스크 완료 시 `TASK_LIST.md` 파일의 해당 체크박스를 `[x]`로 변경
   - 변경 후 사용자에게 완료 보고

4. **진행 상태 보고**
   - 현재 진행 중인 태스크 번호 명시
   - 완료된 태스크 수 / 전체 태스크 수 표시
   - 예: `[3/25] 완료 - 다음 태스크: 4번`

## 태스크 상태

| 상태 | 표시 | 설명 |
|------|------|------|
| 대기 | `[ ]` | 아직 시작하지 않음 |
| 진행중 | `[~]` | 현재 작업 중 |
| 완료 | `[x]` | 작업 완료 |
| 스킵 | `[-]` | 사용자 요청으로 건너뜀 |

## 사용자 명령어

| 명령어 | 동작 |
|--------|------|
| `다음` / `next` / `진행` | 다음 태스크 실행 |
| `스킵` / `skip` | 현재 태스크 건너뛰기 |
| `중단` / `stop` | 작업 중단 |
| `상태` / `status` | 현재 진행 상황 보고 |
| `목록` / `list` | 남은 태스크 목록 표시 |

## 오류 처리

1. 태스크 실행 중 오류 발생 시
   - 오류 내용 보고
   - 사용자에게 재시도/스킵/중단 선택 요청

2. 파일 접근 오류 시
   - 파일 경로 확인
   - 대안 제시
